# Сбор контекста

Раздел `Сбор контекста` нужен, когда у команды уже есть дерево страниц Confluence и его нужно превратить в локальный общий контекст проекта.

## Что подаётся на вход

- ссылка на корневую страницу Confluence
- опциональный ID коллекции
- глубина обхода
- лимит страниц

Доступ используется из раздела `Подготовка окружения`: отдельный логин и пароль на этой странице вводить не нужно.

## Как работает обход

Система открывает корневую страницу через Playwright, читает текст, находит ссылки внутри страницы и обходит их в пределах того же Confluence.

Чтобы не уйти слишком далеко, есть ограничения:

- максимальная глубина: `3`
- максимальное количество страниц: `50`

Практический старт:

- глубина `1`
- лимит `20`

## Куда сохраняется результат

```text
docs/shared-context/confluence_collections/<collection-id>/
  context_index.md
  manifest.json
  pages/
    001_page.md
    002_page.md
```

`context_index.md` — точка входа в собранный контекст.

`manifest.json` — техническая карта коллекции: корневая ссылка, дата сбора, список страниц, ошибки.

`pages/` — нормализованный текст каждой страницы.

## Как передать коллегам

После сбора открой раздел `Обмен контекстом` и опубликуй категорию `Общий контекст`.

Коллеги увидят новый bundle и смогут забрать его к себе кнопкой `Перенести к себе`.

## Когда использовать

Используй `Сбор контекста`, когда нужно подготовить общую базу знаний по системе, интеграции или доменной области.

Для одной конкретной задачи удобнее использовать ссылки прямо в `Подготовке статьи`.
