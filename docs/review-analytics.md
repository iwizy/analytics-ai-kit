# Ревью аналитики

Раздел `Ревью аналитики` нужен для проверки уже готовой статьи как deliverable, а не для генерации нового draft.

## Что можно дать на вход

- ссылку на Confluence
- локальный файл `.md`, `.txt`, `.docx`, `.pdf`

Источники ревью хранятся отдельно:

```text
artifacts/review_sources/<review-id>/
```

## Что делает система

1. забирает статью по ссылке или файлом
2. определяет `FT` или `NFT` автоматически или берёт тип вручную
3. сравнивает текст с шаблоном разделов
4. ищет:
   - отсутствующие обязательные разделы
   - слабые или пустые разделы
   - внутренние противоречия
   - недоработки
5. формирует рекомендации по доработке

## Что получается на выходе

Markdown-отчёт в:

```text
artifacts/analytics_reviews/<review-id>/
```

Отчёт содержит:

- сводку по качеству
- соответствие шаблону
- противоречия
- недоработки
- рекомендации
- чек-лист доработки

## Как пользоваться

1. открой раздел `Ревью аналитики`
2. укажи `Review ID`
3. добавь статью ссылкой или файлом
4. выбери тип документа:
   - `Auto`
   - `FT`
   - `NFT`
5. выбери модель для ревью
6. нажми `Запустить ревью аналитики`

## Когда использовать

Используй `Ревью аналитики`, когда:

- статья уже написана
- нужно проверить её перед согласованием
- хочется получить независимый разбор по шаблону
- нужно второе мнение до отправки в разработку

Не путать с `Gaps` в `Подготовке статьи`:

- `Gaps` — помогает улучшить pipeline по задаче
- `Ревью аналитики` — проверяет уже готовую статью как конечный документ
