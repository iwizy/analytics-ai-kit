# Генерация комплекта документов

Проект больше не ограничивается выбором `FT` или `NFT`.

В разделе `Подготовка статьи` аналитик выбирает, какие документы нужно подготовить: один документ, пресет или полный комплект.

## Каталог

Каталог целей генерации лежит здесь:

```text
docs/templates/catalog.yaml
```

В каталоге описаны:

- `targets` — отдельные документы, которые можно сгенерировать
- `presets` — готовые наборы документов для типовых сценариев

## Доступные пресеты

- `Фича / изменение`
- `Интеграция`
- `Новый микросервис`
- `Релиз`
- `Полный комплект`

## Как работает генерация

Для каждого выбранного документа система:

- берёт соответствующий шаблон из `docs/templates/microservice`
- собирает контекст из `task.md`, вложений, Confluence-страниц и общего контекста
- генерирует отдельный Markdown-файл
- складывает результат в `artifacts/drafts/<task-id>/`

Дополнительно создаются:

- `*_documents_index.md` — оглавление комплекта
- `*_documents_manifest.json` — технический manifest генерации

## Совместимость со старым FT/NFT

Старые кнопки `Analyze`, `Draft`, `Gaps`, `Refine` сохранены как быстрый сценарий.

Новый основной сценарий — выбрать документы галками и нажать `Сгенерировать выбранное`.
