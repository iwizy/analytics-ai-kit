# Обмен контекстом без облака и без git

## Что устанавливаем

Обязательное:

- `Syncthing` на каждую машину аналитика
- `Docker Desktop`
- `Ollama`
- `VS Code`
- `Continue`

Для сравнения конфликтных версий рекомендуем не отдельный diff-инструмент, а встроенный `VS Code Compare`.

## Как устроен обмен

У каждого аналитика проект и модели остаются локально. Между аналитиками синхронизируется только отдельная папка обмена, например:

```text
team-exchange/
  bundles/
```

Внутри `bundles/` система публикует неизменяемые пакеты с `manifest.json` и файлами общего контекста:

- `docs/shared-context`
- `docs/templates`
- `docs/glossary`

## Как настроить

1. Установи `Syncthing` на каждую машину.
2. На обеих машинах создай или выбери одну и ту же логическую папку обмена, например:
   - macOS: `/Users/<user>/team-exchange`
   - Windows: `C:\\team-exchange`
3. Подключи эту папку в `Syncthing` на всех машинах.
4. В `Подготовка окружения` укажи путь к папке обмена.
5. Нажми `Сохранить настройки`.
6. Перезапусти стек через `./start.command`, чтобы Docker смонтировал новую папку обмена внутрь сервиса.

## Как работать дальше

1. Положи общий контекст проекта в локальные папки:
   - `docs/shared-context`
   - `docs/templates`
   - `docs/glossary`
2. На странице `Обмен контекстом` нажми `Опубликовать свои изменения`.
3. У коллег появятся новые bundle-пакеты.
4. Они открывают страницу `Обмен контекстом` и нажимают `Забрать к себе`.

## Что происходит при конфликте

Если локальный файл уже существует и отличается от входящего, система не перезатирает его молча. Вместо этого рядом создаётся incoming-версия:

```text
example.incoming_<bundle-id>.md
```

Рекомендуемый сценарий:

1. открыть локальный файл и incoming-версию в `VS Code Compare`
2. перенести нужные правки
3. удалить incoming-файл после разборки

## Что не синхронизируем

- логины и пароли
- `Playwright` state
- `.continue/config.yaml`
- локальные модели `Ollama`
- кэши, `node_modules`, `.venv`
- временные рабочие артефакты
